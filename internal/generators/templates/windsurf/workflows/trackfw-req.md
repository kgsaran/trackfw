# trackfw: Criar REQ (Requisição)

Guia para criar um documento de requisição antes de qualquer implementação não trivial.

## Quando usar

Para features novas, mudanças em lógica de negócio, alterações de contrato de API ou qualquer mudança não trivial.

## Passos

1. Identifique o que precisa ser implementado.
2. Documente a **Solicitação**: descrição clara do que deve ser feito.
3. Documente o **Escopo**: mudanças específicas, arquivos afetados, restrições.
4. Documente os **Critérios de Aceite**: condições mensuráveis para conclusão.
5. Salve em `docs/requisições/<area>/backlog/REQ-<descricao>-<YYYY-MM-DD>.md`.
6. Crie o roadmap em microlotes baseado nesta REQ.
