# Roadmap v2.2 — Python CLI Nativo

> Criado em: 2026-06-13 | Status: ✅ Done
> REQ: [REQ-2026-06-13-python-cli-nativo](../../../requisições/claude/REQ-2026-06-13-python-cli-nativo.md)

**REGRA DURA:** Toda feature deve ser implementada nos 3 CLIs (Go + Node.js + Python). Nenhum ML conclui sem paridade nos 3.

**Referência de implementação:** `npm/src/` é o espelho de paridade — cada módulo Python tem um equivalente JS.

---

## Diagnóstico

O `pypi/trackfw/_cli.py` atual é um wrapper que baixa o binário Go do GitHub em runtime.
Falha em ambientes corporativos com GitHub bloqueado ou mirror PyPI interno.
O Node.js CLI (`npm/src/`) demonstra o modelo correto: reimplementação nativa, zero dependências externas, stdlib apenas.

**Estrutura alvo:**
```
pypi/trackfw/
├── __init__.py
├── __main__.py          # python -m trackfw
├── cli.py               # entry point (argparse)
├── config.py            # load()/reset() — espelho de npm/src/config/index.js
├── validator.py         # validate() — espelho de npm/src/validator/index.js
├── i18n/
│   ├── __init__.py
│   ├── en-US.json
│   ├── pt-BR.json
│   └── es-ES.json
├── generators/
│   ├── __init__.py
│   ├── adr.py           # espelho de npm/src/generators/adr.js
│   ├── req.py           # espelho de npm/src/generators/req.js
│   ├── roadmap.py       # espelho de npm/src/generators/roadmap.js
│   └── init.py          # espelho de npm/src/generators/init.js
└── commands/
    ├── __init__.py
    ├── adr.py
    ├── req.py
    ├── roadmap.py
    ├── validate.py
    ├── status.py
    ├── log.py
    ├── discover.py
    ├── metrics.py
    ├── context.py
    ├── sync.py
    └── plugins.py
```
