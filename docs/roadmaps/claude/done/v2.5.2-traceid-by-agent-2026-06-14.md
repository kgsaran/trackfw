# Roadmap v2.5.2 — Fix traceid checks em by_agent

> Criado em: 2026-06-14 | Status: ✅ Done

## Diagnóstico

Em projetos com `roadmap_namespacing: by_agent`, os 5 checks `traceid_*` silenciavam
completamente — exit 0 com zero violações mesmo com pares `req_id` quebrados.

**Causa raiz por CLI:**

| CLI | Função | Problema | Status |
|-----|--------|---------|--------|
| Go | `collectTraceIdEntries` | só varria `rootDir/<estado>/`; by_agent tem `rootDir/<agente>/<estado>/` | ✅ Corrigido |
| Python | `_index_roadmaps` | mesmo problema — iterava explicitamente sobre estados sem subir um nível | ✅ Corrigido |
| Node.js | `walkMd` + `stateFromPath` | já funcionava — `walkMd` é recursivo e `stateFromPath` deriva estado do path | ✅ Teste + salvaguarda |

---

## Wave 1 — Fix by_agent + salvaguarda (3 MLs em paralelo)

### ML-1A — Go

**Status:** ✅ Concluído

**Abordagem:**
- Nova função `collectTraceIdEntriesByAgent(roadmapDir, traceField string, cfg config.ProjectConfig)`:
  descobre agentes via `cfg.Agents` (ou `os.ReadDir` se vazio) e varre `roadmapDir/<agente>/<estado>/`
- `validateTraceId` despacha para a nova função quando `cfg.RoadmapNamespacing == config.NamespacingByAgent`
- Salvaguarda: quando `len(reqEntries) == 0 && len(roadmapEntries) == 0`, emite warning de configuração
- 2 novos testes: `TestTraceIdByAgent` e `TestTraceIdZeroEntriesSalvaguarda` — todos 8/8 PASS

### ML-1B — Node.js

**Status:** ✅ Concluído

**Abordagem:**
- Lógica de indexação já correta (`walkMd` recursivo + `stateFromPath` por segmentos)
- Salvaguarda adicionada a `checkTraceIds`: quando `reqIndex.size === 0 && roadmapIndex.size === 0`, retorna `traceid_config_warning`
- Novo arquivo `npm/tests/traceid_by_agent.test.js` com 5 testes: by_agent orphan, par válido, state mismatch, salvaguarda dirs vazios, salvaguarda sem campo — todos 5/5 PASS

### ML-1C — Python

**Status:** ✅ Concluído

**Abordagem:**
- Nova função `_index_roadmaps_by_agent(roadmap_dir, field, agents)`: autodescobre agentes ou usa `cfg["agents"]`
- `check_traceid` despacha para nova função quando `cfg["roadmap_namespacing"] == "by_agent"`
- Salvaguarda: quando `req_entries` e `roadmap_entries` ambos vazios, retorna `traceid_config_warning`
- 2 novos testes: `test_traceid_by_agent` e `test_traceid_zero_entries_warning` — todos 8/8 PASS
