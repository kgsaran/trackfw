# Roadmap v2.5.3 — Fix REQ indexing em by_agent

> Criado em: 2026-06-14 | Status: 🔄 WIP

## Diagnóstico

A v2.5.2 corrigiu o scanner de Roadmaps para `by_agent`, mas o scanner de REQs permanece plano.
Em projetos com `req_dir/<agente>/<estado>/` (ADR-036), todas as checagens de REQ ficam inertes.
No CMDB real: REQs (0), Roadmaps (116) → 15 falsos `traceid_orphan_roadmap`.

**Funções afetadas por CLI:**

| CLI | Funções | Problema |
|-----|---------|---------|
| Go | `validateREQsHaveADR`, `validateREQsHaveRoadmap`, `blockedREQs`, `validateREQsNotBlockedByDraftADRs`, `validateFrontmatterPresence`, `validateRefTargetsExist`, `validateTraceId` | `listDir(cfg.REQDir)` / `filepath.Glob(cfg.REQDir + "/*.md")` — flat |
| Node.js | mesmas 6+ funções | `listDir(cfg.reqDir)` — flat |
| Python | mesmas 6+ funções | `list_dir(cfg["req_dir"])` — flat |

---
