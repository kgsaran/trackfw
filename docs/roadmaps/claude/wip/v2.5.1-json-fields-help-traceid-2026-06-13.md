# Roadmap v2.5.1 — Bugfix --json rule/file + help trace_id_field

> Criado em: 2026-06-13 | Status: 🔄 WIP

## Diagnóstico

Dois bugs complementares identificados na validação de campo da v2.5.0 no CMDB:

1. `validate --json` emite `rule: ""` e `file: ""` em cada item — `applyRule` conhece o
   `ruleName` mas não o propaga ao `RuleItem`; `extractFile` não existe.
2. `trackfw help` não lista `trace_id_field` nem as 5 `rules.traceid_*` adicionadas na v2.5.0.

Ambos são ortogonais entre si e entre CLIs → 3 MLs em paralelo (um por CLI, cada um resolve os 2 bugs).

---

## Wave 1 — Fix --json rule/file + help traceid (3 MLs em paralelo)

> Dependências: independente

### ML-1A — Go

**Status:** ⬜ Pendente  
**Arquivos afetados:**
- `internal/validator/result.go` — `messageToRuleItem` / `applyRule` plumbing
- `internal/validator/validator.go` — `applyRule` propaga ruleName
- `internal/commands/help.go` — adicionar entradas ao `configDocs`
- `internal/commands/validate_json_test.go` — atualizar asserções de `rule`/`file`

**Ações:**
1. Em `result.go`: implementar `extractFile(msg string) string` — extrai primeiro token entre aspas duplas que pareça um path (ex: `"rm.md"` → `rm.md`). Regex: `"([^"]+)"`.
2. Mudar `applyRule` em `validator.go` para receber e propagar `ruleName` ao construir RuleItems, preenchendo `Rule = ruleName` e `File = extractFile(msg)`.
3. Ajustar o pipeline para que `ValidateUnfiltered` e `Validate` retornem `([]RuleItem, []RuleItem, error)` — OU manter `[]string` internamente e só converter no `BuildResult` via uma variante que receba pares `(ruleName, msg)`. Escolha a abordagem menos invasiva.
4. Em `help.go`, adicionar ao slice `configDocs`:
   - `trace_id_field` (string, default `""`, descrição, exemplo `trace_id_field: req_id`)
   - `rules.traceid_orphan_roadmap` (off/warning/error, default `error`)
   - `rules.traceid_orphan_req` (off/warning/error, default `error`)
   - `rules.traceid_state_mismatch` (off/warning/error, default `error`)
   - `rules.traceid_duplicate_req` (off/warning/error, default `error`)
   - `rules.traceid_duplicate_roadmap` (off/warning/error, default `error`)
5. Atualizar `validate_json_test.go` para verificar que `rule != ""` e `file != ""` nos itens.

**Critérios de aceite:**
- [ ] `trackfw validate --json` → itens com `rule` e `file` preenchidos
- [ ] `trackfw help` lista `trace_id_field` e as 5 `rules.traceid_*`
- [ ] `go test ./internal/... -q` verde sem regressões

**Comandos de validação:**
```bash
make build
go test ./internal/commands/ -run TestHelp -v
go test ./internal/commands/ -run TestValidateJSON -v
go test ./... -q 2>&1 | tail -5
```
