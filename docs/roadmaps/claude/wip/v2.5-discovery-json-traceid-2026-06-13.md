# Roadmap v2.5 — Discovery Mode + validate --json + req_id bidirecional

> Criado em: 2026-06-13 | Status: 🔄 WIP

## Diagnóstico / Contexto

Três evoluções identificadas após validação da v2.4.1 no projeto CMDB:

1. **validate --json** (quickest win): saída estruturada para CI/CD pipelines.
2. **Discovery Mode** (maior impacto de adoção): paths configuráveis + namespacing por agente +
   `trackfw discover` — sem isso, projetos com estrutura não-padrão (como o CMDB) não operam.
3. **req_id bidirecional** (rigor de rastreabilidade): verificação REQ↔Roadmap por ID estável.

Ordem de waves: --json é independente (Wave 1). Discovery Mode segue em 3 blocos sequenciais
(Waves 2-4). req_id depende de paths configuráveis do Bloco B (Wave 5).

Paridade obrigatória: Go · Node.js · Python nos 3 CLIs.

> Dependências: independente — pode rodar em paralelo com qualquer outra wave

### ML-1A — validate --json (Go)

**Status:** ⬜ Pendente  
**Arquivos afetados:**
- `internal/commands/validate.go`
- `internal/validator/validator.go` (ou struct de resultado)
- `internal/commands/validate_json_test.go` (novo)

**Ações:**
1. Adicionar flag `--json` ao `newValidateCmd()` em `validate.go`.
2. Criar struct `ValidateResult` em `validator.go` (ou arquivo novo):
   ```go
   type RuleItem struct {
       Rule    string `json:"rule"`
       File    string `json:"file"`
       Message string `json:"message"`
   }
   type ValidateResult struct {
       Summary    ValidateSummary `json:"summary"`
       Violations []RuleItem      `json:"violations"`
       Warnings   []RuleItem      `json:"warnings"`
   }
   type ValidateSummary struct {
       Violations int    `json:"violations"`
       Warnings   int    `json:"warnings"`
       Mode       string `json:"mode"`
       ExitCode   int    `json:"exit_code"`
   }
   ```
3. Quando `--json`: serializar resultado via `encoding/json` e imprimir em stdout; sair com mesmo exit code.
4. Sem `--json`: comportamento atual inalterado.
5. Para extrair `rule` por item: os messages atuais têm formato `"[RULE] file: msg"` ou similar — parsear ou refatorar `applyRule` para retornar struct em vez de string.

**Critérios de aceite:**
- [ ] `trackfw validate --json` emite JSON válido
- [ ] `summary.mode` = `"lenient"` ou `"strict"` conforme config
- [ ] Cada item tem `rule`, `file`, `message`
- [ ] Exit code idêntico ao sem `--json`
- [ ] `go test ./internal/commands/ -run TestValidateJSON` verde

**Comandos de validação:**
